# 🧠 EEG Signal Visualizer & Motor Imagery Classifier

> ⚠️ **The `data/` folder currently ships with SYNTHETIC placeholder data**, not real EEG. It was generated by `generate_synthetic_data.py` so the full pipeline (app UI, CSP+LDA classifier, plots) could be built and tested without network access to physionet.org. Cross-val accuracy on synthetic data is ~95% — that number reflects a clean synthetic signal, not real-world BCI performance, and should not be quoted as a real result. Run `train_model.py` locally (see below) to replace it with a model trained on genuine PhysioNet recordings before using this for a portfolio piece or LinkedIn post.

An interactive Streamlit app that visualizes real EEG recordings and
classifies imagined left-fist vs. right-fist movement using a
CSP + LDA pipeline — the classical, well-validated approach from
brain-computer interface (BCI) literature.

Built on the [PhysioNet EEG Motor Movement/Imagery Dataset](https://physionet.org/content/eegmmidb/1.0.0/)
(64-channel EEG, 109 subjects).

---

## How it works

1. **`train_model.py`** (run once, locally, needs internet access to
   physionet.org) — downloads a handful of real subjects via MNE,
   filters/epochs the signal, trains a CSP+LDA classifier, and saves:
   - `data/csp_lda_model.joblib` — the trained classifier
   - `data/demo_data.npz` — a small set of precomputed epochs for the app to display
2. **`generate_synthetic_data.py`** (already run for you) — produces
   placeholder data in the exact same format, for pipeline testing
   when real data isn't reachable. **Currently what's in `data/`.**
3. **`app.py`** (deployed on Streamlit Cloud) — loads whatever is in
   `data/` and lets you explore waveforms, band power, and live
   predictions. It never downloads or trains anything at runtime, so
   it stays fast on free-tier hosting.

## Setup (local)

```bash
git clone <your-repo-url>
cd eeg-bci-visualizer
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# One-time: download data + train the model (~5-15 min depending on connection)
python train_model.py

# Run the app
streamlit run app.py
```

`train_model.py` downloads data via MNE into a local `mne_data/`
cache the first time it runs — this is normal and only happens once.

## Deploying to Streamlit Community Cloud

1. Run `train_model.py` locally first so `data/csp_lda_model.joblib`
   and `data/demo_data.npz` exist.
2. Commit the entire repo **including** the `data/` folder to GitHub
   (these files are small — a few MB — since the demo set is
   deliberately kept compact).
3. Go to [share.streamlit.io](https://share.streamlit.io), connect
   your GitHub repo, and point it at `app.py`.
4. Done — the deployed app only reads the precomputed files, so it
   starts in seconds and doesn't need MNE's downloader or heavy
   compute at runtime.

## Tech stack

- **MNE-Python** — EEG loading, filtering, epoching
- **scikit-learn** — CSP (via `mne.decoding.CSP`) + LDA classification
- **Streamlit** — UI
- **Plotly** — interactive waveform / band-power charts
